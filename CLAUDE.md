# CLAUDE.md

This file provides guidance to Claude (the AI assistant) when working with this repository.

## Project Overview

`git_practice` is a beginner-friendly Git practice repository. Its purpose is to provide a simple sandbox for learning and experimenting with Git concepts such as commits, branches, merges, and pull requests.

## Repository Structure

```
git_practice/
├── CLAUDE.md      # AI assistant guidance (this file)
├── README.txt     # Project description
├── hello.txt      # Sample hello-world text file
└── test.txt       # Sample test text file
```

## Guidelines for Claude

- **Keep it simple.** This is a practice repo — changes should remain lightweight and easy to understand.
- **Plain text files** (`.txt`, `.md`) are the primary file format used here.
- **No build system or dependencies** — there is nothing to install or compile.
- **No test suite** — this repository is not a software project; avoid adding unnecessary tooling.
- When making changes, prefer small, focused commits with clear commit messages.
- Follow the existing file naming conventions (lowercase, descriptive names).

## Common Tasks

- Adding or editing plain text files to practice staging and committing.
- Creating and switching branches to practice branching workflows.
- Merging branches and resolving conflicts as a learning exercise.
