## Common Tasks

- **Adding files:** `git add <file>` or `git add .` to stage all changes
- **Committing:** `git commit -m "your message"` to commit staged changes
- **Checking status:** `git status` to see the current state of the working tree
- **Viewing history:** `git log --oneline` to see commit history

## Conventions

- Commit messages should be clear and descriptive.
- File names use lowercase with hyphens or dots as separators (e.g., `hello.txt`, `test.txt`).
- Plain `.txt` files are used for content in this practice repo.
